# Markdown Formatting Guidelines Below are the standard rules for formatting content with Markdown: - For headings, use number signs (`#`) before the heading text, the number of `#` corresponds to the heading level (e.g. `#` for level 1 headings, `##` for level 2 headings) - For unordered list items, start each line with a dash (`-`), followed by a space and the list content - To emphasize text, wrap it with asterisks: use single asterisks for *italic text*, double asterisks for **bold text** - For inline code or commands, surround them with backticks, e.g. run `pip install pandas` to install the data analysis library - For quoted text, use greater than signs (`>`) at the start of the line: > This is an example of quoted reference content - For links, wrap the anchor text in square brackets `[]`, followed by the URL in parentheses `()`, e.g. [GitHub Official Site](https://github.com) - For images, add an exclamation mark first, then square brackets `[]` for the alt text, followed by the image URL in parentheses `()`, e.g. ![Markdown Logo](https://upload.wikimedia.org/wikipedia/commons/thumb/4/48/Markdown-mark.svg/1200px-Markdown-mark.svg.png)
