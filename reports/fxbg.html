# Markdown Formatting Specifications Below are the standard Markdown formatting requirements with practical examples for each rule: - For headings, use number signs (`#`): Use 1 to 6 `#` symbols to represent heading levels 1 to 6, e.g. `### Level 3 Heading` - For list items, start with dashes (`-`) for unordered lists, or sequential numbers for ordered lists - To emphasize text, wrap it with asterisks (`*`): *this is emphasized text* is written as `*this is emphasized text*` - For code or commands, surround them with backticks (`` ` ``): Use single backticks for inline code (e.g. `npm install`) and triple backticks for multi-line code blocks - For quoted text, use greater than signs (`>`): > This is a sample quoted block, often used for citing external content or highlighting key notes - For links, wrap the text in square brackets `[]`, followed by the URL in parentheses `()`: e.g. [Markdown Official Guide](https://www.markdownguide.org) is written as `[Markdown Official Guide](https://www.markdownguide.org)` - For images, use square brackets `[]` for the alt text, followed by the image URL in parentheses `()`: e.g. ![Markdown logo](https://upload.wikimedia.org/wikipedia/commons/thumb/4/48/Markdown-mark.svg/1280px-Markdown-mark.svg.png)
