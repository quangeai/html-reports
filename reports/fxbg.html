# Markdown Formatting Reference Guide Below are the standard formatting specifications for creating structured, readable Markdown content: ## Core Formatting Rules - **Headings**: Use number signs (`#`) to define heading hierarchy, with 1 to 6 `#` characters representing H1 to H6 level headings - **Unordered list items**: Start all unordered list entries with a dash (`-`) followed by a space - **Emphasized text**: Wrap text you want to highlight/italicize with single asterisks, for example *this is sample emphasized content* - **Inline code/commands**: Surround short code snippets, terminal commands or technical identifiers with backticks, for example `pip install pandas` - **Block quoted text**: Precede quoted content with a greater than sign (`>`): > This is an example of blockquote content, often used for citations, important notes or highlighted feedback. - **Hyperlinks**: Wrap link display text in square brackets, followed immediately by the target URL in parentheses, for example [Google Search](https://www.google.com) - **Images**: Add an exclamation mark first, then square brackets for image alt text, then the image resource URL in parentheses, for example ![Sample landscape photo](https://picsum.photos/800/400)
